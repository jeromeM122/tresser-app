Tresser — Guide d'intégration PWA

Snippet d'enregistrement du Service Worker + structure de projet complète.

1 Structure des fichiers à la racine

Avant d'intégrer le Service Worker, votre projet doit avoir la structure suivante. Les fichiers en vert sont les nouveaux fichiers PWA générés.

tresser.fr/
├── manifest.json ← fichier PWA 1/2
├── service-worker.js ← fichier PWA 2/2
├── index.html ← votre page Tresser
└── icons/
    ├── icon-192x192.png
    ├── icon-192x192-maskable.png
    ├── icon-512x512.png
    └── icon-512x512-maskable.png
⚠
Important : Le fichier service-worker.js doit être à la racine du domaine (tresser.fr/service-worker.js), pas dans un sous-dossier. Son scope — la zone qu'il contrôle — est limité au répertoire où il se trouve.

2 Liens à ajouter dans le <head>

Ajoutez ces deux balises dans le <head> de votre index.html, juste avant la fermeture </head>. La première lie le manifest PWA, la seconde configure la couleur de la barre de statut iOS.

index.html — <head><!-- PWA : Manifest -->
<link rel="manifest" href="/manifest.json">

<!-- iOS : couleur barre de statut -->
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
<meta name="apple-mobile-web-app-title" content="Tresser">

<!-- iOS : icône d'écran d'accueil (fallback Safari) -->
<link rel="apple-touch-icon" href="/icons/icon-192x192.png">

<!-- Chrome / Android : couleur du thème -->
<meta name="theme-color" content="#1A1A18">

3 Snippet d'enregistrement du Service Worker

Copiez ce bloc JavaScript et collez-le juste avant la balise </body> de votre index.html, après votre script existant.

index.html — avant </body><script>
/**
 * Tresser — Enregistrement du Service Worker
 * À placer juste avant </body>, après les autres scripts.
 */

if ('serviceWorker' in navigator) {

  /**
   * On attend que la page soit complètement chargée (load)
   * avant d'enregistrer le SW. Cela évite de concurrencer
   * le chargement des ressources critiques de la page.
   */
  window.addEventListener('load', async () => {

    try {

      const registration = await navigator.serviceWorker.register(
        '/service-worker.js',
        {
          /* scope : '/' = contrôle tout le domaine */
          scope: '/'
        }
      );

      console.log('[Tresser PWA] Service Worker enregistré ✓',
        registration.scope);

      /**
       * Écouter les mises à jour disponibles.
       * Quand Anthropic déploie une nouvelle version du SW,
       * on notifie l'utilisatrice pour qu'elle puisse rafraîchir.
       */
      registration.addEventListener('updatefound', () => {

        const newWorker = registration.installing;

        newWorker.addEventListener('statechange', () => {
          if (
            newWorker.state === 'installed' &&
            navigator.serviceWorker.controller
          ) {
            /* Une mise à jour est prête — afficher un toast discret */
            showUpdateToast();
          }
        });

      });

    } catch (error) {
      console.error('[Tresser PWA] Échec enregistrement SW :', error);
    }

  });

} else {
  console.info('[Tresser PWA] Service Workers non supportés sur ce navigateur.');
}


/**
 * showUpdateToast()
 * Affiche un bandeau discret en bas de page pour inviter
 * la cliente à rafraîchir et profiter de la nouvelle version.
 */
function showUpdateToast() {
  const toast = document.createElement('div');

  toast.style.cssText = `
    position: fixed;
    bottom: calc(72px + env(safe-area-inset-bottom) + 12px);
    left: 50%;
    transform: translateX(-50%);
    background: #1A1A18;
    color: #FAFAF7;
    padding: 12px 20px;
    border-radius: 100px;
    font-family: 'DM Sans', system-ui, sans-serif;
    font-size: 13px;
    font-weight: 500;
    display: flex;
    align-items: center;
    gap: 12px;
    white-space: nowrap;
    z-index: 9999;
    box-shadow: 0 4px 24px rgba(0,0,0,0.3);
    cursor: pointer;
  `;

  toast.innerHTML = `
    ✦ Mise à jour disponible
    <span style="
      padding: 4px 12px;
      border-radius: 100px;
      background: #C17B4E;
      color: #fff;
      font-size: 11px;
    ">Actualiser</span>
  `;

  toast.addEventListener('click', () => {
    /* Envoie un message au SW pour activer la mise à jour */
    navigator.serviceWorker.controller?.postMessage({ type: 'SKIP_WAITING' });
    window.location.reload();
  });

  document.body.appendChild(toast);
}
</script>
✓
Pourquoi window.addEventListener('load', ...) ? Si on enregistre le SW dès l'exécution du script, il démarrerait son pré-cache en concurrence avec les polices, images et CSS de la page. En attendant l'événement load, la page s'affiche d'abord à pleine vitesse, et le SW démarre ensuite en arrière-plan.
